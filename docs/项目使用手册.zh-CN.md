# GNPR-SID 项目使用手册

本文档面向当前 `GNPR-SID` 新版项目的实际实现，覆盖以下内容：

- 当前项目采用的技术路线
- 目录与关键产物说明
- Linux 服务器上的完整操作流程
- 每一步对应的命令
- 当前实现中的关键设计决策
- 常见问题与排查方法

本文档只描述当前仓库里已经实现并实际跑通过的链路，不描述尚未实现的理想化方案。

## 1. 项目目标

本项目是对旧版 GNPR-SID 的 Linux-first 重构，当前主线流程为：

1. 从旧项目导入 NYC 数据
2. 基于 V2 风格的 `cosine quantization + EMA smoothing` 训练 SID
3. 导出 `pid -> semantic id` 映射
4. 构建 SID-LLM alignment 数据并进行对齐训练
5. 将 alignment adapter 合并回基座模型
6. 在 merged model 上做下一 POI 推荐 SFT
7. 构建 retrieval bank 与 `similar_map_test`
8. 本地离线推理
9. 本地评测与汇总

当前正式支持的数据集：`NYC`

当前正式支持的模型 profile：`Qwen2.5-7B-Instruct`

## 2. 当前实际采用的技术方案

### 2.1 SID 训练

当前 SID 主线采用：

- projection-based cosine quantization
- EMA smoothing
- 三层 codebook
- 输出 token 形式：`<a_x><b_y><c_z>` 或 `<a_x><b_y><c_z><d_w>`

相关配置文件：

- `configs/train/sid_nyc.yaml`

相关代码：

- `src/gnprsid/sid/train.py`
- `src/gnprsid/sid/export.py`
- `src/gnprsid/sid/v2/`

### 2.2 Alignment 训练

当前 alignment 采用：

- `TRL + PEFT`
- 只对 `embed_tokens` 做 LoRA 对齐
- 对齐数据为双向映射：
  - `POI attributes -> SID`
  - `SID -> POI attributes`

相关配置文件：

- `configs/train/alignment_trl.yaml`

相关代码：

- `src/gnprsid/alignment/build_data.py`
- `src/gnprsid/train/base.py`

### 2.3 SFT 训练

当前推荐任务 SFT 采用：

- `LLaMA-Factory`
- 基座模型不是原始 Qwen，而是 `alignment` 后 merge 得到的模型
- 当前默认训练集是：`sid/current`

相关配置文件：

- `configs/train/sft_llamafactory.yaml`

相关代码：

- `src/gnprsid/train/base.py`
- `src/gnprsid/train/merge.py`

### 2.4 Retrieval

当前 retrieval 采用：

- `Qwen2.5-7B-Instruct` 作为 encoder
- `bf16`
- `mean pooling`
- `max_length = 2048`
- 相似度为 cosine similarity

注意：

- `attention pooling` 不是默认方案，因为在前面的实验中已经验证过数值不稳定
- 当前官方产物以 `similar_map_test_{repr}.json` 为主

相关配置：

- `configs/retrieval/default.yaml`

相关代码：

- `src/gnprsid/retrieval/bank.py`
- `src/gnprsid/retrieval/encoder.py`
- `src/gnprsid/retrieval/similarity.py`

### 2.5 本地推理

当前本地推理已经不是“自由生成 10 个候选”，而是：

- prompt 明确要求输出 `1` 个候选
- 但解码时对候选空间做约束
- 使用 constrained beam search 在合法 SID/ID 候选集合中取 top-k
- 最后返回 top10 候选

这样做的原因是：

- 当前 SFT 的监督标签只有 `next POI` 这一个真实标签
- 直接训练“生成 10 个候选”没有严格监督信号
- 因此推理阶段采用 `single-target training + constrained top-k ranking`

这也是当前项目和早期 prompt-only 版本的核心差异之一。

相关代码：

- `src/gnprsid/inference/modeling.py`
- `src/gnprsid/inference/batch.py`

## 3. 目录结构

项目根目录：

```text
GNPR-SID/
  configs/
  data/
  artifacts/
  checkpoints/
  outputs/
  src/
  tests/
  docs/
```

各目录含义：

- `configs/`
  - 所有 YAML 配置
- `data/NYC/`
  - 导入后的原始数据、处理中间文件、样本文件
- `artifacts/NYC/`
  - SID、alignment、retrieval 等中间产物
- `checkpoints/NYC/`
  - SID、alignment、SFT 训练输出
- `outputs/NYC/`
  - 推理预测结果、评测结果、汇总结果
- `src/gnprsid/`
  - 核心代码

## 4. 关键文件说明

### 4.1 模型配置

文件：

- `configs/models/qwen25_7b.yaml`

作用：

- 指定 base model、本地 tokenizer 路径、dtype、max_length、generation 配置

注意：

- 服务器上必须把 `base_model` 和 `tokenizer_name` 改成本地路径
- 如果这里写成 `Qwen/Qwen2.5-7B-Instruct`，离线环境会尝试访问 Hugging Face

### 4.2 SID 配置

文件：

- `configs/train/sid_nyc.yaml`

作用：

- 指定 POI embedding 模型
- 指定 SID 训练超参数
- 指定 SID checkpoint 输出路径

注意：

- `category_model_name` 和 `region_model_name` 目前也通常需要写成服务器本地路径

### 4.3 Alignment 配置

文件：

- `configs/train/alignment_trl.yaml`

作用：

- 指定 alignment 数据路径
- 指定对齐训练 LoRA 参数
- 指定 output_dir

### 4.4 SFT 配置

文件：

- `configs/train/sft_llamafactory.yaml`

关键点：

- `base_model_override: checkpoints/NYC/alignment/qwen25_7b/merged`

这表示：

- SFT 的基座模型不是原始 Qwen
- 而是 alignment adapter merge 后的模型

## 5. 数据与样本说明

### 5.1 旧项目导入

命令：

```bash
python -m gnprsid.cli data import-legacy \
  --dataset NYC \
  --legacy-root /path/to/old/GNPR-SID
```

该步骤会：

- 拷贝旧项目中的 `train/val/test/history/...`
- 将数据放到 `data/NYC/`
- 从旧项目原始 `NYC.txt` 提取真实经纬度
- 补充到 `data/NYC/processed/poi_info.csv`

### 5.2 样本构造

命令：

```bash
python -m gnprsid.cli data prepare-nyc \
  --dataset NYC \
  --current-k 49
```

生成内容包括：

- `data/NYC/processed/samples_id_train.jsonl`
- `data/NYC/processed/samples_sid_train.jsonl`
- `artifacts/NYC/llm/id/...`
- `artifacts/NYC/llm/sid/...`

当前 `current_k` 代表：

- 当前轨迹只保留最近 `K` 个已观测 POI

### 5.3 如果已经有导出的 SID

```bash
python -m gnprsid.cli data prepare-nyc \
  --dataset NYC \
  --current-k 49 \
  --sid-map-path artifacts/NYC/sid/pid_to_sid.json
```

## 6. 完整训练与评测流程

以下流程假设你在 Linux 服务器上运行，并且项目路径为：

```bash
~/proj/GNPR-SID
```

统一先执行：

```bash
cd ~/proj/GNPR-SID
export PYTHONPATH=$PWD/src
export WANDB_MODE=disabled
```

### 6.1 导入旧项目数据

```bash
python -m gnprsid.cli data import-legacy \
  --dataset NYC \
  --legacy-root /path/to/old/GNPR-SID
```

### 6.2 初次准备数据

```bash
python -m gnprsid.cli data prepare-nyc \
  --dataset NYC \
  --current-k 49
```

### 6.3 训练 SID

```bash
python -m gnprsid.cli sid train \
  --config configs/train/sid_nyc.yaml
```

说明：

- 当前 SID 训练没有自动 early stopping
- 实际使用中建议关注 `collision_rate`
- 通常以 `best_collision_model.pth` 为导出依据

### 6.4 导出 SID

```bash
python -m gnprsid.cli sid export \
  --config configs/train/sid_nyc.yaml \
  --checkpoint-path checkpoints/NYC/sid/<run_dir>/best_collision_model.pth
```

导出后会得到：

- `artifacts/NYC/sid/pid_to_sid.json`
- `artifacts/NYC/sid/pid_to_sid.csv`

### 6.5 使用导出的 SID 重建样本

```bash
python -m gnprsid.cli data prepare-nyc \
  --dataset NYC \
  --current-k 49 \
  --sid-map-path artifacts/NYC/sid/pid_to_sid.json
```

### 6.6 构建 alignment 数据

```bash
python -m gnprsid.cli alignment build-data \
  --dataset NYC \
  --sid-map-path artifacts/NYC/sid/pid_to_sid.json
```

输出包括：

- `artifacts/NYC/alignment/semantic_code_mapping.json`
- `artifacts/NYC/alignment/train_align.json`
- `artifacts/NYC/alignment/valid_align.json`

### 6.7 运行 alignment

```bash
python -m gnprsid.cli train run \
  --stage alignment \
  --config configs/train/alignment_trl.yaml
```

输出目录：

- `checkpoints/NYC/alignment/qwen25_7b/`

关键产物：

- `final/`

### 6.8 合并 alignment adapter

```bash
python -m gnprsid.cli train merge-peft \
  --model-config configs/models/qwen25_7b.yaml \
  --adapter-path checkpoints/NYC/alignment/qwen25_7b/final
```

输出目录：

- `checkpoints/NYC/alignment/qwen25_7b/merged`

说明：

- 后续 SFT 与某些对照实验都依赖这个 merged model

### 6.9 运行 SFT

```bash
python -m gnprsid.cli train run \
  --stage sft \
  --config configs/train/sft_llamafactory.yaml
```

输出目录：

- `checkpoints/NYC/sft/qwen25_7b_sid_current/llamafactory_output`

说明：

- 当前默认只对 `sid/current` 做 SFT
- 如果要训练 `original/retrieval/hybrid`，需要先修改 `train_path/valid_path`

## 8. 本地推理流程

### 8.1 推理模式

当前支持：

- `current`
- `original`
- `retrieval`
- `hybrid`

### 8.2 纯 base 模型对照

不传 `--checkpoint-path`：

```bash
python -m gnprsid.cli infer batch \
  --dataset NYC \
  --repr sid \
  --history-source current \
  --model-config configs/models/qwen25_7b.yaml \
  --output-path outputs/NYC/predictions/sid_current_base.json
```

### 8.3 alignment-only 对照

```bash
python -m gnprsid.cli infer batch \
  --dataset NYC \
  --repr sid \
  --history-source current \
  --model-config configs/models/qwen25_7b.yaml \
  --checkpoint-path checkpoints/NYC/alignment/qwen25_7b/merged \
  --output-path outputs/NYC/predictions/sid_current_align_only.json
```

### 8.4 alignment + SFT

```bash
python -m gnprsid.cli infer batch \
  --dataset NYC \
  --repr sid \
  --history-source current \
  --model-config configs/models/qwen25_7b.yaml \
  --checkpoint-path checkpoints/NYC/sft/qwen25_7b_sid_current/llamafactory_output \
  --output-path outputs/NYC/predictions/sid_current_test.json
```

### 8.5 retrieval 模式

```bash
python -m gnprsid.cli infer batch \
  --dataset NYC \
  --repr sid \
  --history-source retrieval \
  --model-config configs/models/qwen25_7b.yaml \
  --checkpoint-path checkpoints/NYC/sft/qwen25_7b_sid_current/llamafactory_output \
  --retrieval-bank-path artifacts/NYC/retrieval/retrieval_bank_sid.jsonl \
  --similar-map-path artifacts/NYC/retrieval/similar_map_test_sid.json \
  --top-k-retrieval 5 \
  --output-path outputs/NYC/predictions/sid_retrieval_test.json
```

### 8.6 当前推理实现要点

当前本地推理有几个重要实现细节：

1. `infer batch` 现在带进度条  
   终端会显示 `Generating`

2. `eval run` 终端只打印 metrics  
   完整样本仍保存在 json 文件中

3. 如果 checkpoint 是 PEFT adapter  
   推理会优先读取 `adapter_config.json` 中的 `base_model_name_or_path`

4. 当前 top10 不是自由生成  
   而是 constrained ranking

## 9. 评测流程

### 9.1 单次评测

```bash
python -m gnprsid.cli eval run \
  --predictions outputs/NYC/predictions/sid_current_test.json \
  --output-path outputs/NYC/eval/eval_sid_current.json
```

当前终端只显示：

- `acc_at_1`
- `acc_at_5`
- `acc_at_10`
- 以及若干辅助统计

json 文件里仍然保留：

- `metadata`
- `metrics`
- `samples`

### 9.2 汇总评测

```bash
python -m gnprsid.cli eval summarize \
  --dataset NYC
```

## 10. 当前已知的实现边界

以下不是 bug，而是当前版本的设计边界：

1. `GRPO` 仍然只是接口预留，未实现完整训练链路  
2. 当前 SFT 默认只训练 `sid/current`
3. 当前 top10 是 constrained ranking，不是“多标签生成式 SFT”
4. retrieval 官方默认使用 `mean pooling`，不是 attention pooling
5. 当前正式支持的数据集只有 `NYC`

## 11. 常见问题

### 11.1 本地推理时报 Hugging Face 网络错误

原因：

- `configs/models/qwen25_7b.yaml` 里写的是 HF 模型名
- 离线环境会尝试联网拉模型

解决：

- 把 `base_model` 和 `tokenizer_name` 改为服务器本地目录

### 11.2 `eval run` 输出太长

当前已处理：

- 终端只显示 `metrics`
- 详细样本保存在输出 json

### 11.3 本地推理出现 generation flags warning

当前已处理：

- 推理入口会规范化 generation config
- `do_sample=False` 时不会再使用无效的采样参数

### 11.4 SFT 推理结果杂乱

这是前期版本的问题，已处理：

- 修正了 left padding 下的输出截取逻辑
- 改为 constrained top-k ranking

## 12. 推荐实验顺序

如果你要做最小闭环验证，建议顺序如下：

1. `data import-legacy`
2. `data prepare-nyc`
3. `sid train`
4. `sid export`
5. `data prepare-nyc --sid-map-path`
6. `alignment build-data`
7. `train run --stage alignment`
8. `train merge-peft`
9. `train run --stage sft`
10. `retrieval build-bank`
11. `retrieval build-similar --split test`
12. `infer batch`
13. `eval run`

如果只做对照实验，建议至少跑这三组：

- `base`
- `alignment only`
- `alignment + SFT`

## 13. 当前最重要的结论

当前项目已经满足以下条件：

- 代码层面可以完整跑通
- Linux 服务器上可以从旧项目导入数据并完成新链路训练
- 可以生成 SID、alignment、SFT、retrieval、local inference、eval 全流程产物
- 本地推理与评测逻辑已经闭合

但需要明确：

- 当前性能表现不等于最终论文效果
- 如果后续要继续提升 top10 输出质量，主方向应该是更强的训练目标或 GRPO，而不是回到自由生成 10 个标签的 SFT
